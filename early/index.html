The Early History of UNIX

UNIX's inspiration came from Multics - an early time-sharing system developed from 1964 — 1969 by MIT, General Electric and Bell Labs. Multics is actually an acronym, standing for MULTIplexed Iformation and Computing Services. Bell Labs withdrew from the Multics project in 1969, and the Bell Labs developers working on the project were without something to do, Ken Thompson and Dennis Ritchie among them. Thompson and Ritchie began work on another operating system to bide their time - a replacement for Multics on the compact PDP-7 mainframe computer. One wise-cracking colleague suggested it be called the UNiplexed Iformation and Computing Services; it was similar to Multics, but with less power and for smaller machines. UNICS, later changed to UNIX, was the name that stuck.

Did you know?: UNIX was first developed to host a game that Thompson wrote called "Space Travel" that was originally written for the GE-645, but development was hampered by its price ($75/hour!). A dusty PDP-7 laying around was picked up by Thompson, and then saw the need for a compact operating system for the PDP-7 in order to run his game.

UNIX is based upon a more simple and modular philosophy that was very different from other mainframe systems of the time. Thompson and Ritchie aimed to create an operating system based around a large collection of software tools that perform one task, as opposed to monolithic applications that handle a variety of tasks and may share the same internal functions. Another core part of the philosophy was piping - using the output of one program as the input of another - an example is using the commands that print the text content of a file (cat file.txt) and finds all lines that start with A (grep -E "^A") and piping the inputs as so: cat file.txt | grep -E "^A".

The first real test of UNIX came when the text formatting utilities of the system were beginning to be used by AT&T's patent department. It was a huge success for the patent examiners and the UNIX developers, as they now had greater credibility to purchase a more powerful PDP-11. All these new programs required a new form of documentation; The UNIX Programmer's Manual was published on the 3rd of November 1971, and included command documentation written in the "man page" format — terse, descriptive summaries of the commands, their arguments, what they do, and the programmer's contact details; all interactively provided inside UNIX itself by typing man <command>. As UNIX was in constant development, many people simply described the version of UNIX as the version of The UNIX Programmer's Manual that was written for it.

The first edition of UNIX had a number of innovative features, one in particular that we all now take for granted — a hierarchical file system. This meant that files could now be sorted into directories, which could also contain other directories. At the time, file names could only be six characters large, but this was seen as plenty enough. Another innovative idea was to rewrite the entirety of UNIX in C, a high-level programming language. Operating systems were previously written only in assembly code, which was non-portable, verbose and very hard to remix and develop. Solving that problem by expressing it in a language that many considered didn't offer the control that operating systems required was a big deal at the time, and UNIX derivatives are still written in C today.
